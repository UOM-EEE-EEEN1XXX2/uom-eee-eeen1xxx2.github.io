1.1. Course aims¶

This unit aims to provide students with an introduction to programming computer systems. It will provide an understanding of software development tools, and hands on experience of coding simple programs to build practical skills. This will give awareness of different types of programming approach (interpreted, procedural, and object orientated), and an introduction to coding for problem solving and creating algorithms. The unit focuses on the programming of desktop/laptop class devices, with some consideration of high performance computing (whether for high-spec computers, or embedded/single-board computer systems).

1.2. Topics¶

The unit will cover the following topics:

  1. Introduction and motivation (including considering factors such as sustainability, risk, security, and equality/diversity/inclusion in programming).

  2. Overview of major operating systems (e.g. Windows, macOS, Linux, Real-time Operating Systems (RTOS)).

  3. Software engineering techniques (e.g. integrated development environments, version control, software lifecycle, debugging, unit testing).

  4. General automation of computer tasks (e.g. shell scripting).

  5. Core programming concepts (e.g. variables, if statements, loops; illustrated in Python).

  6. Computer hardware and software concepts for programmers (e.g. memory addressing, heap and stack memory, data types, number formats, ASCII and UFT-8).

  7. Practice using interpreted languages, including using standard packages and performing data visualization (e.g. via Python, Numpy, Scipy, Pandas/Polars, Matplotlib).

  8. Practice using compiled high-performance languages (e.g. Rust).

  9. Awareness of other compiled high-performance languages (e.g. C/C++).

  10. Overview of coding for problem solving: common algorithms and high performance computing concepts.

  11. Industry relevance (e.g. guest lecture).

  12. Revision.

Items 0-9 are covered in these notes. Items 10 and 11 are only covered in the in-person lectures, if taking the course in-person in Manchester.

1.3. Intended learning outcomes¶

After successfully completing the course students should be able to:

  1. Explain the need for, and importance of, programming, including why higher performance code may be needed and how this can be achieved. This includes an awareness of factors such as sustainability and security which may affect program design and implementation.

  2. Demonstrate an awareness of the hardware and software in a computer system and how to program a desktop/laptop type device.

  3. Explain the software development process, from requirements capture to long-term support, including the importance of documentation.

  4. Use standard development tools (e.g. integrated development environments and version control) for writing and debugging software.

  5. Apply standard programming techniques (e.g. if statements, loops) in different programming languages.

  6. Test developed software to give evidence that the code is working and is functionally correct.

  7. Perform data visualization using standard libraries.

  8. Work individually to translate a given descriptive problem or algorithm into program code using appropriate programming elements, techniques and data structures.
